# F.0 — Hello Grasshopper

*(Nguồn: Grasshopper Primer 3rd Edition — Mode Lab, trang PDF 15–34)*

Đây là chương mở đầu, giới thiệu Grasshopper là gì, cách cài đặt/khởi động, giao diện (UI), và cách
Grasshopper "nói chuyện" với Rhino (baking, units, remote control panel...).

Định nghĩa mở đầu của sách: Grasshopper là 1 trình soạn thảo thuật toán dạng đồ họa (graphical
algorithm editor), tích hợp với công cụ modeling của Rhino3D. Dùng Grasshopper để thiết kế thuật
toán, rồi thuật toán đó tự động hoá các thao tác trong Rhino3D.

---

## F.0.0 Installing & Launching Grasshopper

### F.0.0.0 Downloading
Vào website Grasshopper, bấm tab **Download**, nhập email khi được hỏi. Chuột phải vào link tải,
chọn **Save Target As** để lưu file cài đặt về ổ cứng máy — sách lưu ý KHÔNG tải/chạy được qua kết
nối mạng (network connection), file phải nằm sẵn trên ổ cứng local.

### F.0.0.1 Installing
Chạy **Run** từ hộp thoại tải về, làm theo hướng dẫn của trình cài đặt (Installation Wizard). Điều
kiện bắt buộc: máy phải cài **Rhino 5** sẵn thì plugin mới cài đúng được. Sách nhắc: Grasshopper cập
nhật thường xuyên nên luôn cập nhật bản mới nhất; **không có bản Grasshopper cho Mac** (tại thời
điểm sách viết).

### F.0.0.2 Launching
Gõ lệnh **`Grasshopper`** vào dòng lệnh (command line) của Rhino để khởi chạy. Cửa sổ Grasshopper sẽ
nổi (floating) đè lên trên các viewport của Rhino. Sách khuyên nên sắp 2 cửa sổ không đè lên nhau
(kéo mỗi cửa sổ về 1 bên màn hình, hoặc giữ phím Windows + phím mũi tên trái/phải để dán cửa sổ sang
2 nửa màn hình). Số phiên bản Grasshopper hiện đang cài được hiển thị ở thanh trạng thái (status bar)
phía dưới cửa sổ.

## F.0.1 The Grasshopper UI

Grasshopper là ứng dụng lập trình trực quan (visual programming): người dùng tạo **definitions**
(cũng gọi là **documents** — tức là 1 "chương trình" Grasshopper) bằng cách kéo-thả **components**
vào vùng làm việc chính gọi là **canvas**. Output của component này nối vào input của component
tiếp theo, tạo thành 1 đồ thị (graph) thông tin đọc từ trái sang phải.

Sách liệt kê các thành phần chính của giao diện (đọc từ trên xuống, kèm mô tả trình tự bố trí):
- **Windows title bar** — thanh tiêu đề cửa sổ.
- **Main menu bar** — thanh menu chính (File, Edit, View, Display, Solution...).
- **File browser control** — nằm bên phải thanh menu, hiển thị lưới 3×3 các file gần đây.
- **Component palettes** — vùng chứa các tab/panel component.
- **Canvas** — vùng làm việc chính ở giữa.
- **Canvas toolbar** — thanh công cụ nhỏ phía trên canvas.

Ô lưới 3×3 (file browser control) hiển thị các file mở gần đây nhất theo thứ tự thời gian; nếu 1
file không tìm thấy nữa (bị di chuyển/xoá) sẽ hiện ô hình chữ nhật màu đỏ.

### F.0.1.0 The Windows Title Bar
Thanh tiêu đề của cửa sổ Editor hoạt động khác với dialog thông thường của Windows: nếu cửa sổ
không đang minimize/maximize, **double-click vào thanh tiêu đề** sẽ thu gọn (collapse) dialog thành
1 thanh nhỏ — cách nhanh để chuyển qua lại giữa Grasshopper và Rhino mà không cần di chuyển cửa sổ
hay đưa nó xuống dưới cùng. Lưu ý: đóng cửa sổ Editor thì phần preview hình học trong Rhino viewport
biến mất, nhưng file KHÔNG thực sự đóng — chạy lại lệnh `Grasshopper` sẽ mở lại đúng trạng thái/file
cũ, vì phiên (session) Grasshopper vẫn sống cho tới khi Rhino đóng hẳn.

### F.0.1.1 Main Menu Bar
- Menu **File**: các chức năng thường gặp (New File, Open, Save...) + công cụ xuất ảnh (**Export
  Quick Image**, **Export Hi-Res Image**).
- Menu **View** và **Display**: điều khiển các khía cạnh khác nhau của giao diện.
- Menu **Solution**: quản lý các thuộc tính về cách solver tính toán kết quả đồ thị.
- **Preferences** (trong menu File) chứa hầu hết cài đặt ứng dụng: mục **Author** (thông tin cá nhân
  lưu kèm mỗi file GH), mục **Display** (tùy chỉnh giao diện chi tiết), mục **Files** (tần suất/vị
  trí lưu autosave), mục **Solver** (quản lý plugin lõi và plugin bên thứ 3).
- Cảnh báo của sách: cẩn thận khi dùng phím tắt (shortcuts) vì phím tắt được xử lý bởi cửa sổ đang
  active (có thể là Rhino, canvas Grasshopper, hoặc 1 cửa sổ khác trong Rhino) — dễ gõ nhầm lệnh vì
  tưởng đang ở cửa sổ này nhưng thực ra đang active ở cửa sổ khác.
- 2 loại file Grasshopper: **`.gh`** (file dữ liệu nhị phân, mặc định) và **`.ghx`** (Grasshopper
  XML — dùng thẻ tag để định nghĩa object và thuộc tính, giống HTML nhưng tag riêng của GH; vì là
  file text nên có thể mở bằng NotePad để xem code).

### F.0.1.2 File Browser Control
Cho phép chuyển nhanh giữa các file đang mở qua danh sách xổ xuống (drop-down). Click vào tên file
active trong ô browser sẽ hiện danh sách xếp tầng (cascading list) của mọi file đang mở, kèm ảnh
thumbnail nhỏ của từng definition. Phím tắt **Alt+Tab** cũng chuyển nhanh giữa các file GH đang mở.
Có thể mở file qua hộp thoại Open File chuẩn, hoặc **kéo-thả file GH thẳng vào canvas**.

Sách liệt kê 5 cách mở file khác nhau khi kéo-thả vào canvas (right-click / kéo thả sẽ hỏi chọn 1
trong các chế độ này):
- **Open File**: mở definition được kéo-thả vào canvas (như bình thường).
- **Insert File**: chèn 1 file có sẵn vào document hiện tại dưới dạng các component rời rạc (loose).
- **Group File**: chèn file vào document hiện tại, nhưng gom tất cả object lại thành 1 **Group**.
- **Cluster File**: tương tự Group File, nhưng tạo thành 1 **Cluster object** bao trọn cả nhóm.
- **Examine File**: mở file ở trạng thái khoá (locked) — chỉ xem được, không sửa được definition.

Grasshopper có tính năng **Autosave** kích hoạt định kỳ dựa theo hành động người dùng (cấu hình ở
menu File). Khi đóng Rhino, 1 hộp thoại pop-up sẽ hỏi có muốn lưu các file GH đang mở không. Lưu ý:
Autosave chỉ hoạt động nếu file đã được lưu (save) ít nhất 1 lần trước đó.

### F.0.1.3 Component Palettes
Khu vực này tổ chức component theo danh mục (categories, hiển thị dạng **tab**) và danh mục con
(sub-categories, hiển thị dạng **panel xổ xuống — drop-down panel**). Mọi component đều thuộc 1
category nào đó (vd "Params" cho các kiểu dữ liệu cơ bản, "Curves" cho công cụ liên quan đường
cong). Có 2 cách thêm component vào canvas: **click** vào icon trong menu xổ xuống, hoặc **kéo-thả**
trực tiếp từ menu vào canvas.

Vì mỗi sub-category có thể chứa nhiều component hơn số ô hiển thị được, panel chỉ hiện 1 số lượng
icon giới hạn. Có thể chỉnh **chiều cao của component palette** và **chiều rộng cửa sổ Grasshopper**
để hiện nhiều/ít icon hơn mỗi panel. Để xem TOÀN BỘ component trong 1 sub-category, click vào **thanh
đen ở đáy mỗi panel** — mở ra menu dropdown chứa đủ mọi component thuộc sub-category đó.

Trình tự bố trí (từ trên/ngoài vào trong, theo mô tả sách): **Category tab** (thanh tab trên cùng,
click vào thanh đen dưới để mở panel con) → **Sub-category panel** (khối icon) → **Drop-down menu**
(khi bấm thanh đen ở đáy panel, hiện đủ danh sách). Di chuột (hover) lên 1 component sẽ hiện mô tả
ngắn (tooltip).

### F.0.1.4 The Canvas
Canvas là vùng làm việc chính để tạo definition — nơi đặt component và nối dây (wires). Đây là nơi
tương tác trực tiếp với các phần tử của chương trình trực quan.

### F.0.1.5 Grouping
Nhóm (group) các component lại giúp dễ đọc, dễ hiểu định nghĩa hơn, và cho phép chọn/di chuyển
nhiều component cùng lúc. Cách tạo group: chọn các component muốn nhóm rồi gõ **Ctrl+G**. Cách khác:
nút **"Group Selection"** trong menu **Edit** của Main Menu Bar. Right-click vào 1 group để chỉnh
màu (color), độ trong suốt (transparency), tên (name), và kiểu viền (outline type).

Có 2 kiểu viền (outline profile) được nhắc tới: **Box Outline** (khung hộp vuông) và **Blob Outline**
(dùng thuật toán meta-ball, tạo viền cong kiểu giọt). Sách minh hoạ 1 ví dụ: 2 group lồng nhau, group
ngoài đổi màu xanh nhạt để phân biệt; các group được vẽ "phía sau" các component bên trong nó, và có
thứ tự lớp (depth order) — muốn đổi thứ tự này thì vào **Edit > Arrange** ở main menu bar.

### F.0.1.6 Widgets
Các widget bật/tắt qua menu **Display** của Main Menu Bar. Sách giới thiệu 3 widget:
- **The Align Widget**: chọn nhiều component cùng lúc, sau đó click 1 nút trong khung viền nét đứt
  bao quanh vùng chọn. Có thể căn trái, căn giữa dọc, căn phải, căn trên, căn giữa ngang, căn dưới,
  hoặc **phân bố đều (distribute)** các component. Sách lưu ý: mới học dễ dùng lộn khiến các
  component chồng lên nhau, nhưng quen tay thì rất hữu ích để giữ đồ thị gọn gàng dễ đọc.
- **The Profiler Widget**: liệt kê thời gian tính toán tệ nhất (worst-case runtime) của từng
  parameter/component — giúp tìm điểm nghẽn (bottleneck) hiệu năng. Widget này **mặc định TẮT**.
- **The Markov Widget**: dùng chuỗi Markov (Markov chain — 1 quy trình gồm hữu hạn trạng thái với
  xác suất chuyển đổi đã biết) để "dự đoán" component nào bạn có thể muốn dùng tiếp theo, dựa trên
  hành vi trước đó. Cần thời gian để widget "làm quen" với người dùng. Gợi ý tối đa **5 component**
  tùy hoạt động gần đây. Vị trí mặc định: **góc dưới-trái canvas**; right-click để dock sang góc
  khác hoặc ẩn hẳn.

### F.0.1.7 Using the Search Feature
**Double-click vào khoảng trống trên canvas** sẽ mở 1 ô tìm kiếm (search box) pop-up — gõ tên
component cần tìm, danh sách parameter/component khớp sẽ hiện ra.

### F.0.1.8 The Find Feature
Khác với search feature (tìm để THÊM component mới), **Find** dùng để tìm 1 component **đã có sẵn
trên canvas**. Cách gọi: **right-click bất kỳ đâu trên canvas** hoặc nhấn phím **F3**. Gõ tên
component cần tìm. Find feature tìm cả theo tên chính thức (title — tên hiển thị trong Component
Panel, người dùng không đổi được) lẫn **nickname** (biệt danh do người dùng tự đặt cho component),
và có thể tìm trong nội dung text panel, scribble, group. Khi tìm thấy, GH sẽ làm mờ (grey out) phần
còn lại của definition và vẽ khung nét đứt quanh component khớp. Nếu có nhiều kết quả khớp, danh
sách hiện trong hộp thoại Find, hover vào 1 item sẽ tô xanh (green) component tương ứng trên canvas.
Mỗi item trong danh sách có 1 mũi tên nhỏ trỏ tới vị trí component trên canvas — kéo hộp thoại Find
đi thì mũi tên tự xoay theo để giữ hướng trỏ đúng.

Ví dụ minh hoạ của sách: tìm từ khoá "divide" → liệt kê ra nhiều component khớp, gồm **Division**
(toán tử chia), **Divide Surface**, **Divide Domain2**.

### F.0.1.9 Using the Radial Menu
Gọi **radial menu** (menu hình tròn xoay quanh 1 tâm) bằng cách nhấn **phím cách (Space bar)** khi
chuột đang ở trên canvas hoặc trên 1 component, hoặc **click chuột giữa (middle mouse button)**.
Radial menu hiện tính năng khác nhau tùy vào việc gọi trên 1 component đang chọn hay chỉ gọi trên
khoảng trống canvas — gọi trên component sẽ có nhiều lựa chọn hơn.

### F.0.1.10 The Canvas Toolbar
Thanh công cụ nhỏ nằm phía trên canvas, cho truy cập nhanh các tính năng hay dùng (mọi tính năng ở
đây cũng có trong menu). Có thể ẩn/hiện lại qua tab **View** trên Main Menu Bar. Các nút được liệt
kê:
- **Open File / Save File**: mở/lưu nhanh.
- **Zoom Defaults**: các mức zoom mặc định định sẵn.
- **Zoom Extents**: zoom vừa khít toàn bộ definition; bấm vào mũi tên cạnh icon để chọn thêm zoom
  tới 1 vùng cụ thể.
- **Named Views**: lưu/gọi lại 1 vùng nhìn (view area) cụ thể trong definition.
- **The Sketch Tool**: công cụ vẽ phác thảo, hoạt động tương tự công cụ bút chì trong Adobe
  Photoshop, có thêm vài tính năng riêng. Cho phép chỉnh line weight, line type, màu. Right-click
  vào object sketch có thể **simplify** (làm mượt đường vẽ), hoặc chọn **"Load from Rhino"** để nạp
  1 hình 2D có sẵn trong Rhino thay cho nét vẽ tay — chọn hình 2D trong Rhino scene rồi nhấn Enter,
  nét sketch cũ sẽ được cấu hình lại theo đúng hình tham chiếu đó. Lưu ý của sách: sketch object có
  thể bị dịch chuyển khỏi vị trí gốc sau khi nạp shape từ Rhino, vì GH đặt sketch object tương đối
  theo gốc tọa độ canvas (góc trên-trái) và gốc mặt phẳng xy thế giới trong Rhino.
- **Preview Settings**: 3 nút bật/tắt kiểu hiển thị preview toàn cục — **Shaded preview** (mặc
  định), **Wire-frame preview**, **Turn off preview**.
- **Preview Selected Objects**: khi bật, GH chỉ hiện hình học thuộc các component **đang được chọn**,
  kể cả khi các component đó có preview=off.
- **Document Preview Settings**: mở hộp thoại đổi màu mặc định của preview (mặc định: xanh lá bán
  trong suốt cho object được chọn, đỏ bán trong suốt cho object không chọn).
- **Preview Mesh Quality**: chỉnh chất lượng mesh/bề mặt hiển thị trong Rhino — chất lượng cao hơn
  thì thời gian tính toán lâu hơn, thấp hơn thì hiển thị kém chính xác hơn. Lưu ý: các cài đặt này
  chỉ ảnh hưởng đến **hiệu năng hiển thị**, hình học khi **bake vào Rhino** vẫn giữ độ phân giải cao.

## F.0.2 Talking to Rhino

Khác với 1 file Rhino, 1 **definition Grasshopper không chứa object/hình học thật** — nó chỉ đại diện
cho 1 tập luật & chỉ dẫn (rules & instructions) để Rhino tự động thực hiện công việc.

### F.0.2.0 Viewport Feedback
Mọi hình học tạo ra từ component Grasshopper sẽ hiện (mặc định) trong Rhino viewport. Đây chỉ là
**xấp xỉ Open GL** của hình học thật — vì vậy **không chọn (select) được** hình học này trong Rhino
viewport cho tới khi **bake** nó vào scene. Có thể bật/tắt preview của 1 component bằng right-click →
chọn nút toggle **Preview**. Bảng màu mặc định (color coding) để phân biệt trạng thái:
- **Xanh lá (green)**: hình học thuộc component đang được **chọn**.
- **Đỏ (red)**: hình học thuộc component **không được chọn**.
- **Xanh dương (blue)**: đang thực hiện thao tác chọn (selection) trong Rhino Viewport.
- Điểm (point) được vẽ dưới dạng dấu **cộng (cross)** thay vì hình chữ nhật, để phân biệt với point
  object gốc của Rhino.

Bảng màu này chỉnh được qua công cụ **Document Preview Settings** trên canvas toolbar.

### F.0.2.1 Live Wires
Grasshopper là môi trường "sống" (dynamic): mọi thay đổi trong definition đều live — preview trong
Rhino viewport tự cập nhật ngay theo thay đổi.

### F.0.2.2 Gumball Widget
Khi hình học được lưu dạng **internalized** (nội bộ hoá, tức lưu thẳng trong parameter thay vì tham
chiếu ra Rhino) trong 1 parameter Grasshopper, **Gumball** (widget kéo/xoay/scale quen thuộc của
Rhino) cho phép thao tác trực tiếp với hình học đó ngay trong Rhino viewport — cập nhật live theo
từng thao tác kéo Gumball. Ngược lại, hình học **tham chiếu (referenced) trực tiếp từ Rhino** vẫn tồn
tại trong file Rhino, và Grasshopper chỉ cập nhật SAU KHI có thay đổi xảy ra (không phải trong lúc
đang kéo).

### F.0.2.3 Baking Geometry
Muốn thao tác được với hình học Grasshopper trong Rhino (chọn, sửa, transform...), phải **"bake"**
nó. Bake sẽ tạo ra hình học mới thật sự trong Rhino document, dựa theo trạng thái hiện tại của đồ thị
Grasshopper — hình học đó sau khi bake **không còn phản hồi theo thay đổi của definition** nữa (đã
"đóng băng" thành object Rhino độc lập).

Cách bake: right-click vào component → chọn **Bake**. Hoặc nhấn phím **INSERT** để bake tất cả
component đang chọn vào layer Rhino hiện đang active. Một hộp thoại sẽ hiện ra để chọn layer Rhino
nào sẽ nhận hình học được bake. Sách gợi ý: nên **gom nhóm (group)** hình học vừa bake để dễ quản lý,
đặc biệt khi tạo rất nhiều object bằng Grasshopper.

### F.0.2.4 Units & Tolerances
Grasshopper **kế thừa đơn vị (units) và dung sai (tolerances) từ Rhino**. Để đổi đơn vị: gõ lệnh
**`Document Properties`** vào dòng lệnh Rhino để mở menu Document Properties, chọn mục **Units**.

### F.0.2.5 Remote Control Panel
**Remote Control Panel (RCP)** là 1 giao diện tối giản để điều khiển definition mà không chiếm nhiều
diện tích màn hình — hữu ích khi làm việc trên 1 màn hình duy nhất (thay vì phải liên tục zoom/di
chuyển cửa sổ Grasshopper). Bật RCP qua nút toggle trong menu **View** của Main Menu Bar. Mặc định
RCP trống rỗng.

Để đưa 1 phần tử UI (slider, toggle, button...) vào RCP: right-click phần tử đó, chọn **"Publish To
Remote Panel"** — thao tác này tạo 1 group mới và đồng bộ phần tử UI đó vào RCP. Đổi giá trị trong
RCP sẽ đồng thời cập nhật giá trị trong đồ thị chính và mọi hình học phụ thuộc vào tham số đó. Có thể
publish nhiều phần tử để tạo 1 giao diện điều khiển hoàn chỉnh, không cần hiện toàn bộ đồ thị trực
quan đè lên Rhino viewport.

RCP có **2 chế độ**: **Working Mode** (mặc định — chỉnh giá trị thật của các phần tử UI) và **Edit
Mode** (chỉnh layout — vị trí, tên, màu của các nhóm). Chuyển sang Edit Mode bằng cách click vào
**cây bút chì màu xanh lá** ở góc trên-phải RCP; nền RCP sẽ đổi sang **màu cam** khi đang ở Edit
Mode. Trong Edit Mode có thể: tạo nhóm UI mới, sắp xếp lại phần tử trong nhóm, thêm nhãn (label),
đổi màu. Xoá 1 phần tử UI: kéo nó ra ngoài viền RCP. **Không đổi được giá trị tham số khi đang ở Edit
Mode** — phải bấm lại bút chì xanh để về Working Mode.

Lưu ý sách nêu: RCP sẽ tự lấy **tên của phần tử UI** làm nhãn hiển thị — nên đặt tên slider/toggle rõ
nghĩa trước khi publish, vì tên đó sẽ hiện thẳng trong RCP.

### F.0.2.6 File Management
Nếu 1 file Grasshopper tham chiếu (reference) hình học từ Rhino, bắt buộc phải mở đúng file Rhino đó
thì definition mới chạy đúng. Sách khuyên: giữ file Grasshopper và file Rhino **trong cùng 1 thư
mục project**, đặt tên liên quan tới nhau để dễ quản lý.

### F.0.2.7 Templates
Có thể tạo và chỉ định 1 file **template** trong Preferences của Grasshopper, để mỗi definition mới
tạo ra tự động dùng sẵn cấu trúc đó — tiện lợi khi luôn cần vài component/panel/sketch nhất định
(vd panel ghi chú, sketch object đánh nhãn) trong mọi file mới. Các bước theo sách: (1) tạo 1 file
chứa sẵn các component/panel mong muốn rồi lưu lại làm file template; (2) vào **File > Preferences**,
nạp file đó vào mục **Template File** — từ đó mỗi lần tạo file mới, Grasshopper sẽ tự dùng template
này.
